CS 4530 Summer School · VS Code Extension · Open Source

ADR Manager — TC Annotation Layer

view →

I was the data layer person on a 5-track team building a VS Code extension — the part nobody sees but everyone depends on. Owned Track B: TC Data Model & Storage. Designed the TypeScript interfaces the whole team built against, wired up the YAML read/write, and spent a non-trivial amount of time making sure a grammar migration from one team didn't silently break mine.

schema design Designed TcAnnotation and all enum types (TcCategory, TcSignal, TcStatus) as the shared contract across the full team — if I got this wrong, everyone's work broke
read / write Built parseTcFromYaml() and serializeTcToYaml() — automatically extracts and persists annotation fields from YAML frontmatter without corrupting anything else in the file
validation Built validateTcAnnotation() — validates all 6 field types and returns per-field error messages the UI layer consumes directly
testing 78 tests across 4 suites — and fixed all the fixture breakage when Track A migrated the grammar mid-project, which was its own adventure
TypeScript js-yaml ANTLR4 Jest VS Code Extension API

CookYourBooks

view →

Recipe management app — store recipes, scale servings, step through instructions while you cook. I worked full-stack on several features. The one I'm most proud of: the save function in the recipe editor took me multiple days to get right. Not because it was technically exotic, but because I wanted it to actually work correctly, not just appear to.

feature Built the live timer as a first-class instruction field — editable in the recipe editor, persisted in the data model, starts/pauses and counts down live in cook mode
UI Cook mode step-through interface with active timer — users move through instructions one at a time with the timer ticking alongside
full-stack Full recipe editor: GUI form state, validation, and persistence — the part I debugged for days and would do again
architecture Introduced ThemeViewModel and refactored MainViewController to decouple theme state cleanly; dark mode PR merged via conflict resolution
