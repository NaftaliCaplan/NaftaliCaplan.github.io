CS 4530 Summer School - VS Code Extension - Open Source

ADR Manager - TC Annotation addition

marketplace → repo →

I was the data layer person on a 5-track team building a VS Code extension for managing Architectural Decision Records. Designed the TypeScript interfaces the whole team built against, wired up the YAML read/write, and spent an unreasonable amount of time making sure a grammar migration from another track didn't silently break mine. I then transitioned into a role that required me to merge all tracks, polish any bugs, and extend a couple of features to publish a well developed product.

  • Designed TcAnnotation and all enum types (TcCategory, TcSignal, TcStatus) as the shared contract across the full team
  • Built parseTcFromYaml() and serializeTcToYaml(), reads and writes annotation fields from YAML frontmatter without corrupting anything else in the file
  • Built validateTcAnnotation(), validates all 6 field types and returns per-field error messages consumed by the UI layer
  • Mode-aware serialisation: pro-only fields only written when professional editor is active
  • Schema version stamping and real-world compatibility testing against JabRef's 60-file ADR corpus
  • 82 tests across 4 suites, including fixing all fixture breakage when Track A migrated the grammar mid-project
